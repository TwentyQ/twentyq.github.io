# Лабораторная работа №4: Классификация с применением Scikit-Learn

## Цель работы
Построить и проанализировать модели классификации для предсказания дефолта заёмщика (просрочка более 90 дней) на основе анкетных данных. Оценить качество моделей с использованием метрик (ROC‑AUC, матрица сопряжённости, точность), исследовать влияние порога классификации и попытаться улучшить результат с помощью альтернативных алгоритмов (SVM, kNN).

## Задание
### Основная часть
1. Загрузить данные для обучения и тестирования. 
2. Выполнить предобработку: заполнить пропуски средними значениями, отделить целевую переменную. 
3. Обучить базовые модели: логистическую регрессию и случайный лес.
4. Проверить качество на тестовой выборке: построить confusion matrix, ROC‑кривую, вычислить ROC‑AUC.

### Самостоятельная часть
1. Исследовать другие модели классификации: SVM и kNN.
2. Сравнить полученные значения ROC‑AUC с логистической регрессией и случайным лесом.
3. Проанализировать причины, по которым невозможно достичь ROC‑AUC ≥ 0.9 на данном наборе данных.
4. Изучить современные подходы и дать рекомендации по улучшению.

## Бонус: интеграция обученной модели с веб-сервисом (Flask / FastAPI / Django)

**Пошаговый алгоритм:**

1. Сохранение модели. Экспорт обученной модели в файл (.pkl или .joblib) с помощью joblib.dump().

2. Выбор фреймворка.

3. Создание проекта. Установка зависимостей (fastapi, uvicorn, joblib, pandas/numpy).

4. Загрузка модели при старте. Глобальная переменная model = joblib.load('model.pkl') вне обработчиков.

5. Создание endpoint /predict. POST-запрос, принимает JSON с признаками, преобразует в массив, вызывает model.predict(), возвращает {"prediction": цена}.

6. Обработка ошибок. проверка наличия полей, типов данных; возврат HTTP 400 при ошибке.

7. Локальное тестирование. запуск сервера, отправка тестового запроса через curl или Postman.

8. Развёртывание. выгрузка кода на GitHub, подключение к облачному сервису (Render, Railway, Hugging Face Spaces).

9. Документирование. описание формата запроса и ответа.
