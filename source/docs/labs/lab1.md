# Лабораторная работа №1: Создание и развертывание статического сайта на MkDocs

## Цель работы

Освоить процесс создания статического сайта с использованием генератора документации MkDocs, научиться организовывать структуру портфолио лабораторных работ, изучить базовые принципы работы с Git и GitHub, развернуть сайт на GitHub Pages, а также освоить настройку темы оформления и конфигурационного файла `mkdocs.yml`.

---

## Задание

### Основная часть
1. Создать публичный репозиторий на GitHub для размещения сайта-портфолио.
2. Настроить GitHub Pages для публикации из каталога `/docs` ветки `main`.
3. Клонировать репозиторий на локальный компьютер.
4. Создать и активировать виртуальное окружение Python.
5. Установить MkDocs в виртуальное окружение.
6. Настроить файл `.gitignore`.
7. Создать новый сайт командой `mkdocs new source`.
8. Запустить локальный сервер для предпросмотра.
9. Выполнить сборку сайта в каталог `/docs`.
10. Выполнить коммит и отправку в удалённый репозиторий.

### Самостоятельная часть
1. Выбрать и подключить тему оформления (Material).
2. Настроить файл `mkdocs.yml`:
   - название сайта;
   - тема оформления с розово-черной цветовой схемой;
   - структура меню через параметр `nav`.
3. Создать структуру страниц:
   - Главная страница;
   - Страница «Об авторе»;
   - Раздел «Лабораторные работы» с отдельными страницами;
   - Шаблон структуры отчета.
4. Обеспечить корректную навигацию по сайту.
5. Выполнить повторную сборку и публикацию.

## Код и команды

### 1. Создание репозитория
Создаем репозиторий на GitHub по примеру https://github.com/username/username.github.io.git.
В моем случае: https://github.com/TwentyQ/twentyq.github.io

### 2. Настройка виртуального окружения и установка MkDocs
```bash
# Создание виртуального окружения
python -m venv venv

# Активация (Windows)
venv\Scripts\activate

# Активация (Mac/Linux)
source venv/bin/activate

# Установка MkDocs
pip install mkdocs
pip install mkdocs-material  # для темы Material(в моем случае)
```

### 3. Создание файла .gitignore
```bash
# Виртуальное окружение
venv/
env/
ENV/

# Файлы Python
__pycache__/
*.py[cod]
*.so
.Python

# Редакторы
.vscode/
.idea/
*.swp
*.swo

# Системные файлы
.DS_Store
Thumbs.db
```

### 4. Создание структуры сайта
```bush
# Создание нового проекта в папке source
mkdocs new source

# Переход в папку source
cd source

# Запуск локального сервера для проверки
mkdocs serve
```

### 5. Настройка mkdocs.yml
```bash
site_name: Сайт-портфолио|Данилова Анастасия
nav:
  - Главная: index.md
  - Об авторе: about.md
  - Лабораторные работы:
    - Обзор: labs/index.md
    - Лабораторная работа №1: labs/lab1.md
  - Контакты: contact.md

theme:
  name: material
  palette:
    - scheme: slate
      primary: pink
      accent: deep purple
      toggle:
        icon: material/weather-night
        name: Светлая тема?
    - scheme: default
      primary: pink
      accent: deep purple
      toggle:
        icon: material/weather-sunny
        name: Темная тема?
  
  features:
    - navigation.tabs
    - navigation.top
    - navigation.footer
    - content.code.copy
    - content.code.annotate
  
  hide:
    - navigation #спрятать навигацию слева

markdown_extensions: #для подсветки кода и улучшенной работы с блоками кода
  - pymdownx.highlight:
      anchor_linenums: true
  - pymdownx.superfences
```

### 6. Сборка сайта в каталог /docs
```bash
# Находясь в папке source, выполняем сборку
mkdocs build -d ../docs
```

### 7. Коммит и публикация
```bash
# Вернемся в корневую папку репозитория
cd .. #в моем случае twentyq.github.io

# Добавляем все файлы
git add .

# Делаем коммит
git commit -m "Первая версия сайта-портфолио"

# Отправляем на GitHub
git push origin main
```
